KEY INSIGHT (Mike's Request)
════════════════════════════════════════════════════════════════════════════════════════════════

"Buffer sufficient data to permit real-time function."

✓ We buffer data (RollingTickWindow: 300 ticks = 5 min context)
✓ But learning is real-time (RealtimeDQNLearner: updates per tick)

The buffer provides CONTEXT (for board analysis).
The real-time learning provides ADAPTATION (for DQN weights).

These are separate systems working in parallel:
• DQN learns from tick stream (online, incremental)
• Buffer maintains price history (for decision context)
• Board combines fresh DQN + recent history for confident decisions

Result: DQN learns at market speed. No replay. No compression. Real-time evolution.
